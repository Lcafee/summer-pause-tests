# گزارش کنترل کیفیت

تاریخ اجرا: `2026-07-13`

## خلاصه

نسخه نهایی به‌صورت یک پروژه استاتیک موبایل‌محور بازسازی شد. رابط بصری قبلی، CSS انباشته، ساختارهای قدیمی منو و قابلیت اشتراک‌گذاری ناقص حذف شدند. داده‌ها از رابط جدا شده‌اند و سایت بدون Build، API یا Backend اجرا می‌شود.

## ایده و سیستم طراحی

ایده مرکزی:

> **Printed Summer Cuts — برش‌های چاپی یک مکث تابستانی**

تصمیم‌های اصلی:

- تصویر محصول مرکز ثقل تمام صفحات است.
- محصول کامل در Hero و Result قابل‌شناسایی باقی می‌ماند.
- Fragmentهای تصویری فقط برای ایجاد عمق و ریتم کلاژ استفاده شده‌اند.
- چهار لاین مانند چهار Chapter یک کاتالوگ هنر معاصر طراحی شده‌اند.
- تایپوگرافی فارسی نقش اصلی و لاتین نقش ادیتوریال مکمل دارد.
- رنگ هر نتیجه از شخصیت همان محصول استخراج شده است.
- Motion براساس مونتاژ قطعات، Mask Reveal و Cut طراحی شده است.

## عرض‌ها و حالت‌های تست‌شده

- `320×568`
- `360×640`
- `375×667`
- `390×844`
- `412×915`
- `430×932`
- Landscape در `844×390`
- نمایشگر کوتاه و بلند
- `prefers-reduced-motion: reduce`

نتیجه:

- در هیچ عرض تست‌شده‌ای سرریز افقی مشاهده نشد.
- در نمایشگرهای کوتاه، صفحه شروع به‌جای هم‌پوشانی عناصر، به اسکرول کوتاه کنترل‌شده تبدیل می‌شود.
- Safe Area بالا و پایین در CSS لحاظ شده است.

## مسیرهای اصلی کوییز

تمام هشت مسیر اصلی تست شدند:

| انتخاب سؤال اول | انتخاب سؤال دوم | نتیجه |
|---|---|---|
| خوش‌عطر و شفاف | آرام و گرم | `IT-01` |
| خوش‌عطر و شفاف | پرانرژی و ملس | `IT-02` |
| میوه‌ای و سرزنده | پرطراوت و استوایی | `RF-02` |
| میوه‌ای و سرزنده | معطر و متفاوت | `RF-03` |
| ابری و لطیف | لوکس و اصیل | `CL-01` |
| ابری و لطیف | تضاد لطافت و تردی | `CL-03` |
| برفی و شیرین | هارمونیک و آشنا | `FR-01` |
| برفی و شیرین | گرم و جنوبی | `FR-02` |

نتیجه تست:

- هر هشت نتیجه اصلی صحیح بودند.
- تصویر اصلی هر هشت نتیجه بارگذاری شد.
- پیشنهاد ثانویه در همه تست‌ها با نتیجه اصلی متفاوت بود.
- دکمه تولید پیشنهاد ثانویه تازه، پیشنهاد جدید و غیرهمسان تولید کرد.
- بازگشت از سؤال دوم به سؤال اول درست کار کرد.
- شروع دوباره کوییز درست کار کرد.

## منوی کامل

- چهار Chapter موجود و قابل دسترسی‌اند.
- چهار دکمه ناوبری به Chapter صحیح منتقل می‌شوند.
- هر هشت محصول نمایش داده می‌شوند.
- هر هشت تصویر منو با موفقیت بارگذاری شدند.
- خروج از منو، کاربر را به صفحه قبلی برمی‌گرداند.
- نام، کد، قیمت، توضیح و تگ‌های هر محصول در DOM موجود و قابل خواندن‌اند.

## دسترس‌پذیری و Keyboard

- شروع کوییز با کلید `Enter` تست شد.
- Focus بعد از ورود به کوییز روی Heading سؤال اول قرار می‌گیرد.
- Focus هنگام ورود به سؤال دوم روی Heading سؤال دوم قرار می‌گیرد.
- Focus بعد از Reveal روی نام نتیجه قرار می‌گیرد.
- Focus هنگام پرش میان Chapterها روی Heading همان Chapter قرار می‌گیرد.
- کلید `Escape` برای خروج از Quiz، Result و Menu پشتیبانی می‌شود.
- حالت کاهش حرکت به نتیجه کامل و قابل استفاده منتهی شد.

## تصاویر

ورودی جدید شامل هشت تصویر اصلی `1254×1254` بود. برای هر محصول تولید شد:

- `480px WebP`
- `960px WebP`
- `480px JPEG`
- `960px JPEG`

آمار کل:

- حجم تصاویر اصلی: حدود `14.52 MB`
- مجموع WebPهای مورد استفاده: حدود `419 KB`
- نسبت مجموع WebPها به تصاویر اصلی: حدود `2.9%`
- تصاویر اصلی بدون حذف در `assets/images/source/` نگهداری شده‌اند.
- هیچ Resize غیریکنواخت یا تغییر نسبت لیوان انجام نشده است.
- ابعاد تصویر در HTML مشخص شده تا از Layout Shift جلوگیری شود.
- تصاویر پایین صفحه Lazy Load هستند.

## فونت‌ها

سه نقش تایپوگرافی با Font Stackهای سیستم پیاده‌سازی شدند:

- فارسی و رابط: `Tahoma / Arial`
- لاتین ادیتوریال: `Georgia / Times New Roman`
- جزئیات چاپی: `Arial Narrow / Arial`

هیچ فونت خارجی، CDN یا فایل فونت بدون مجوز وارد پروژه نشده است.

## خطاها و Fallbackها

- در مسیرهای تست‌شده خطای Console ثبت نشد.
- هیچ درخواست Asset شکست‌خورده در تست منو ثبت نشد.
- Fallback تصویر مفقود تست و نمایش داده شد.
- Fallback قیمت نامعتبر تست و مقدار `۳۵۰٬۰۰۰` نمایش داده شد.
- فایل‌های JavaScript با `node --check` بررسی شدند.
- مسیرهای پروژه Relative هستند.

## محدودیت محیط تست

سیاست امنیتی مرورگر Sandbox این محیط، ناوبری مستقیم به آدرس `file://` و `localhost` را مسدود می‌کند. بنابراین تست مرورگری با همان فایل‌های نهایی و یک Runtime استاتیک محلیِ Route‌شده انجام شد. ساختار پروژه Build ندارد، تمام مسیرها Relative هستند و کد از ES Module یا API مرورگر وابسته به HTTP استفاده نمی‌کند؛ در نتیجه `index.html` برای اجرای مستقیم و GitHub Pages آماده است.

جزئیات ماشینی تست در فایل زیر موجود است:

```text
tests/qa-results.json
```
