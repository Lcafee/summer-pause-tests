# L Cafe — Find Your Summer Pause

بازطراحی کامل کمپین تابستانی `L Cafe` با جهت هنری **Contemporary Art Collage** و ایده مرکزی:

> **Printed Summer Cuts — برش‌های چاپی یک مکث تابستانی**

این پروژه یک سایت استاتیک، موبایل‌محور و آماده انتشار روی `GitHub Pages` است. هیچ بک‌اند، API، فریم‌ورک یا Build Step لازم ندارد.

## اجرای محلی

ساده‌ترین روش:

1. پوشه پروژه را باز کنید.
2. فایل `index.html` را در مرورگر اجرا کنید.

روش پیشنهادی برای تست شبیه محیط انتشار:

```bash
python -m http.server 8000
```

سپس در مرورگر باز کنید:

```text
http://localhost:8000
```

## انتشار روی GitHub Pages

1. تمام محتوای همین پوشه را داخل شاخه موردنظر Repository قرار دهید.
2. تغییرات را Commit و Push کنید.
3. در مسیر `Settings > Pages`، همان Branch و پوشه را به‌عنوان Source انتخاب کنید.
4. فایل `.nojekyll` داخل پروژه باقی بماند.

تمام مسیرها Relative هستند و پروژه برای انتشار در Root یا Subdirectory سازگار است.

## محل اطلاعات محصولات

منبع اجرایی اطلاعات سایت:

```text
assets/js/data.js
```

در این فایل می‌توان موارد زیر را تغییر داد:

- نام فارسی و انگلیسی
- توضیح محصول
- تگ‌ها
- قیمت
- رنگ‌های اختصاصی محصول
- اطلاعات چهار لاین
- متن سؤال‌ها و گزینه‌ها
- نگاشت هر مسیر کوییز به نتیجه اصلی

فایل زیر فقط مرجع محتوایی ورودی پروژه است و مستقیماً در Runtime خوانده نمی‌شود:

```text
content/original-content-reference.xls
```

## تغییر قیمت‌ها

در `assets/js/data.js` محصول را با `code` پیدا کنید و مقدار `price` را تغییر دهید:

```js
{
  code: 'IT-01',
  price: 350000
}
```

قیمت به تومان و با اعداد فارسی نمایش داده می‌شود. اگر مقدار قیمت حذف یا نامعتبر شود، مقدار جایگزین `defaultPrice` استفاده خواهد شد.

## تصاویر اصلی و نسخه‌های بهینه‌شده

تصاویر اصلی باکیفیت:

```text
assets/images/source/
```

نسخه‌های مورد استفاده سایت:

```text
assets/images/products/
```

برای هر محصول چهار فایل وجود دارد:

```text
<code>-480.webp
<code>-960.webp
<code>-480.jpg
<code>-960.jpg
```

مرورگر ابتدا `WebP` را انتخاب می‌کند و `JPEG` نقش Fallback را دارد. تمام تصاویر نسبت `1:1` دارند و بدون تغییر نسبت لیوان تولید شده‌اند.

جزئیات ابعاد، حجم و Hash همه تصاویر در این فایل ثبت شده است:

```text
assets/images/image-manifest.json
```

## جایگزینی عکس یک محصول

برای نمونه، برای جایگزینی `FR-02`:

1. فایل اصلی جدید را بدون تغییر نسبت محصول در این مسیر قرار دهید:

```text
assets/images/source/fr-02.png
```

2. چهار نسخه زیر را با همان نام‌ها خروجی بگیرید:

```text
assets/images/products/fr-02-480.webp
assets/images/products/fr-02-960.webp
assets/images/products/fr-02-480.jpg
assets/images/products/fr-02-960.jpg
```

3. ابعاد هر نسخه باید دقیقاً مربع و برابر عرض نام فایل باشد.
4. نسبت لیوان را با Stretch یا Resize غیریکسان تغییر ندهید.

## منطق کوییز

کوییز دقیقاً دو سؤال دارد:

1. سؤال اول، یکی از چهار لاین را مشخص می‌کند.
2. سؤال دوم، یکی از دو محصول همان لاین را به‌عنوان نتیجه اصلی انتخاب می‌کند.

هشت مسیر نتیجه اصلی در `assets/js/data.js` تعریف شده‌اند و هر مسیر به کد محصول قطعی خود متصل است.

### پیشنهاد ثانویه

پیشنهاد ثانویه **عمداً تصادفی** است:

- به مقدار `alt` فایل مرجع متصل نیست.
- هرگز با محصول اصلی یکسان نمی‌شود.
- دکمه «انتخاب تازه» پیشنهاد دیگری تولید می‌کند.
- تا جای ممکن پیشنهاد تازه با پیشنهاد ثانویه قبلی نیز متفاوت است.

## تایپوگرافی

برای جلوگیری از وابستگی خارجی و مسائل مجوز، پروژه از Font Stackهای محلی سیستم استفاده می‌کند و فایل فونت باینری همراه پروژه نیست:

- فارسی و رابط: `Tahoma / Arial / sans-serif`
- تیترهای لاتین ادیتوریال: `Georgia / Times New Roman / serif`
- جزئیات چاپی: `Arial Narrow / Arial / sans-serif`

سایت به `Google Fonts`، CDN یا سرویس فونت خارجی وابسته نیست.

## دسترس‌پذیری

- `lang="fa"` و `dir="rtl"`
- ساختار Heading معنایی
- کنترل‌های واقعی `button`
- Focus State واضح
- جابه‌جایی Focus هنگام تغییر سؤال و صفحه
- اعلان مراحل برای Screen Reader
- Progress معنایی دو مرحله‌ای
- پشتیبانی از Keyboard و کلید `Escape`
- پشتیبانی کامل از `prefers-reduced-motion`
- متن جایگزین تصاویر
- Fallback برای تصویر و قیمت مفقود
