# L Cafe — Find Your Summer Pause

نسخه‌ی نهایی موبایل‌محور و بهینه‌سازی‌شده برای انتشار مستقیم روی GitHub Pages.

ظاهر، محتوا و رفتار نسخه‌ی تأییدشده حفظ شده‌اند. تغییرات این نسخه فقط فنی هستند: کاهش حجم بارگذاری، پاک‌سازی کد، انتخاب هوشمند اندازه‌ی تصاویر و بهبود مسیر اجرای مرورگر.

## اجرای محلی

از داخل پوشه پروژه اجرا کنید:

```bash
python -m http.server 8000
```

سپس این آدرس را باز کنید:

```text
http://localhost:8000
```

فایل `index.html` نیز بدون سیستم Build و بک‌اند قابل اجرا است.

## انتشار روی GitHub Pages

تمام محتویات همین پوشه را در ریشه‌ی مخزن قرار دهید. سپس در بخش `Settings > Pages`، انتشار از شاخه‌ی اصلی و مسیر `/` را فعال کنید.

هیچ Build Step برای انتشار لازم نیست؛ فایل‌های Production از قبل ساخته شده‌اند.

## ساختار پروژه

```text
index.html                         صفحه اصلی و ساختار Semantic
styles/main.css                    CSS خوانا و قابل‌ویرایش
styles/main.min.css                CSS نهایی مورد استفاده سایت
content/catalog.js                 داده محصولات، قیمت‌ها و منطق کوییز
scripts/app.js                     منطق خوانا و قابل‌ویرایش سایت
scripts/site.min.js                باندل نهایی Catalog + App
assets/products/                   تصاویر اصلی 1254×1254
assets/products/responsive/        نسخه‌های بهینه 480/640/960/1100
assets/fonts/                      فونت ادیتوریال محلی و مجوز آن
tools/build.py                     بازسازی فایل‌های Minify
requirements-dev.txt               وابستگی‌های فقط زمان توسعه
QA_REPORT.md                       گزارش کنترل کیفیت و عملکرد
optimization-metrics.json          اعداد خام مقایسه قبل و بعد
```

## تغییر اطلاعات محصولات

نام، کد، قیمت، توضیحات، تگ‌ها، مسیر تصاویر و نگاشت کوییز فقط در فایل زیر تغییر می‌کنند:

```text
content/catalog.js
```

پس از تغییر کد یا CSS، فایل‌های Production را دوباره بسازید:

```bash
python -m pip install -r requirements-dev.txt
python tools/build.py
```

## منطق کوییز

کوییز دقیقاً دو سؤال دارد. سؤال اول لاین را مشخص می‌کند و سؤال دوم محصول اصلی را انتخاب می‌کند.

پیشنهاد ثانویه **عمداً تصادفی** است، از محصول اصلی متفاوت می‌ماند و منطق آن نباید به فیلد `alt` فایل مرجع متصل شود.

## تصاویر

مرورگر براساس عرض صفحه و تراکم پیکسلی، مناسب‌ترین فایل را از `srcset` انتخاب می‌کند:

- صفحه اول: نسخه‌های کوچک‌تر و سریع‌تر
- نتیجه و منوی کامل: نسخه‌های 960 یا 1100 برای موبایل‌های پرتراکم
- فایل 1254 فقط در جایی دریافت می‌شود که واقعاً برای وضوح لازم باشد

## فونت‌ها

`Cormorant Garamond` به‌صورت محلی و Subset شده داخل پروژه قرار دارد. `Estedad` برای حفظ دقیق وزن متغیر و ظاهر فعلی از Google Fonts دریافت می‌شود و Font Stack جایگزین در CSS تعریف شده است.

برای حذف تنها وابستگی خارجی باقی‌مانده، فایل قانونی `Estedad Variable WOFF2` لازم است؛ جایگزینی با وزن‌های Static ممکن است شکست خطوط و ظاهر را اندکی تغییر دهد.
